# Q版桌宠功能设计

## 1. 桌宠定位

Q版桌宠不是独立功能页，而是角色陪伴体验的增强层。它用于：

- 强化角色存在感
- 给聊天增加情绪反馈
- 成为后续付费皮肤、动作、成长系统的承载点

## 2. 当前 mock 实现

当前阶段桌宠由前端组件 `PetWidget` 实现：

- 使用 mock 图片资产
- 接收角色桌宠信息和动作状态
- 通过 CSS 动画表现简单情绪
- 聊天接口根据 mock 回复返回 `pet_action`

## 3. 后续升级为 AI 生成 Q版形象

后续可扩展为：

1. 基于角色设定生成形象提示词
2. 调用图像生成模型输出 Q版角色立绘
3. 生成多表情、多动作资源包
4. 将生成结果存入角色资产库
5. 结合用户编辑形成可迭代形象

## 4. 后续升级为 Live2D / Spine / 3D

### Live2D

- 适合轻量、二次元风格、表情丰富
- 可绑定眨眼、嘴型、头部轻动

### Spine

- 适合 2D 骨骼动画
- 适合做更多动作切换和组合

### 3D

- 可增强沉浸感
- 需要更高的建模、绑定、实时渲染成本

建议演进顺序：

`静态 mock 图 → 多状态 2D 图 → Live2D / Spine → 3D`

## 5. pet_action 状态设计

| action | 含义 | 当前表现 |
| --- | --- | --- |
| `idle` | 默认陪伴 | 上下轻浮动 |
| `happy` | 开心 | 轻微放大 / 跳动 |
| `shy` | 害羞 | 半透明 / 轻微缩小 |
| `thinking` | 思考 | 思考气泡 |
| `talking` | 说话 | 对话气泡 |
| `comfort` | 安慰 | 向输入框方向靠近 |
| `surprised` | 惊讶 | 快速抖动 |
