# 长期记忆系统设计

## 1. 为什么需要长期记忆

Chat Engine 的短上下文只能覆盖最近几轮对话，无法长期保存用户偏好、称呼、情绪状态和关系约定。长期记忆系统用于把可复用的信息落库，并在后续聊天时注入 Prompt，让角色在自然场景下表现出“记得用户”的连续性。

当前第 6 阶段仍不接真实 LLM 记忆抽取，记忆候选来自规则抽取。规则只在用户明确陈述偏好、称呼、情绪或互动信息时生成记忆；疑问句不会被当作偏好事实保存。

## 2. Memory 数据模型

- `memory_id`：记忆业务 ID
- `character_id`：所属角色
- `memory_type`：记忆类型
- `content`：记忆内容
- `importance`：重要度，1 到 5，默认 3
- `source`：来源
- `is_active`：是否有效
- `created_at`：创建时间
- `updated_at`：更新时间

## 3. memory_type 说明

- `user_preference`：用户偏好，例如喜欢、讨厌、称呼
- `relationship`：关系记忆，例如约定、称呼、关系连续性
- `story_interaction`：剧情互动记忆，例如上次发生过的互动
- `emotional_state`：情绪状态，例如难过、疲惫、想哭
- `custom`：用户自定义记忆

## 4. importance 说明

`importance` 用于控制记忆排序和是否自动保存。聊天自动抽取时，当前只自动保存重要度大于等于 3 的候选记忆。

## 5. source 说明

- `chat_auto`：聊天自动提取
- `user_manual`：用户手动添加
- `system_mock`：系统初始 mock

## 6. is_active 软删除设计

删除记忆时不物理删除记录，而是设置 `is_active=false`。软删除记忆不参与 Prompt 注入，也不参与 active memory 去重；如果用户后来再次表达同一内容，可以重新生成一条 active memory。

## 7. MemoryExtractor 当前规则

当前使用本地规则，不调用真实 LLM：

- 用户明确陈述“我喜欢 / 我讨厌 / 我不喜欢 / 我希望 / 我想要”，生成 `user_preference`
- 如果句子是在询问偏好，例如“你记得我喜欢什么吗？”、“我喜欢什么？”、“你知道我喜欢什么吗？”、“我是不是喜欢雨夜散步？”，即使包含“喜欢”，也不生成 `user_preference`
- 用户说“我今天很难过 / 我很累 / 我想哭 / 我不开心”，生成 `emotional_state`
- 用户说“以后叫我 / 你可以叫我 / 我的名字是”，生成 `user_preference`
- 用户提到“我们 / 你还记得 / 上次 / 约定 / 答应”，生成 `relationship` 或 `story_interaction`
- 没有明显可记忆内容时，不生成候选

## 8. 记忆如何注入 Prompt

PromptBuilder 新增 `memory_prompt`，按类型整理 active memories，并加入系统规则：

- 可以参考长期记忆维持关系连续性
- 不要逐条复述记忆
- 只有在自然合适时体现“我记得”

`prompt_debug` 会返回 `active_memory_count` 和 `active_memories_preview`，便于开发调试。

## 9. 如何避免重复记忆

自动保存前会在当前 `character_id` 范围内查找完全相同 `content` 的 active memory。若已存在，则不重复保存，并在 `memory_debug.candidates` 中标记 `saved=false` 和原因。

## 10. 用户如何编辑 / 删除记忆

记忆页读取真实后端记忆，按类型分区展示。用户可以新增、编辑内容/类型/重要度，或“移除记忆”。编辑接口推荐提交 `memory_content` 字段，`content` 仅作为兼容字段。移除记忆调用软删除接口，不再参与后续 Prompt。

## 11. 后续如何升级为 LLM 记忆抽取

后续可将当前 `MemoryExtractor` 替换为真实模型抽取，但保持输出结构不变：

```json
{
  "memory_type": "user_preference",
  "content": "用户喜欢雨夜散步。",
  "importance": 3,
  "reason": "用户明确表达偏好"
}
```

这样 Chat Engine、数据库模型、前端记忆管理页都不需要推翻。
