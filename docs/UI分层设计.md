# UI 分层设计

## 1. 我的数字人首页

1. 顶部品牌区：项目名、slogan、设置入口
2. 内容区：最近陪伴列表
3. 卡片区：头像、角色名、用户身份、关系、最近消息、桌宠小图、亲密度
4. 操作区：右下角悬浮加号按钮

## 2. 创建数字人页面

1. 标题说明区
2. 步骤进度区
3. 上传资料区
4. 文件元信息区
5. 解析结果区：状态、摘要、字数
6. 角色信息表单区
7. 生成按钮区

## 3. 生成结果页

1. 结果说明区
2. 角色信息卡
3. 用户身份与关系摘要区
4. Q版桌宠展示区
5. 已使用解析文档区
6. 操作按钮区

## 4. 角色详情页

1. 角色信息卡
2. 用户扮演身份卡
3. 关系 / 亲密度卡
4. Q版桌宠展示区
5. 操作按钮区

## 5. 聊天页

1. 角色头部信息
2. 当前用户扮演身份提示
3. 消息流
4. 悬浮桌宠
5. 底部输入区

## 6. 记忆页

1. 页面说明区
2. 用户偏好记忆区
3. 角色关系记忆区
4. 剧情互动记忆区
5. 新增记忆输入区

## 7. 设计原则

- 每个页面只承载一个主任务
- 不把“创建、聊天、设置、记忆”混在同一屏
- 桌宠贯穿主流程，但不抢占核心操作
- 通过卡片、分区、留白建立层级
