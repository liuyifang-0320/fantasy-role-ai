# 角色自定义与调教配置设计

## 1. 为什么需要角色自定义

同一个剧本角色在不同用户心中的理想表现可能不同：有人希望回复更温柔，有人希望更克制，有人希望称呼更亲密，也有人希望严格防剧透。角色自定义配置用于在不推翻原角色设定的前提下，让用户调整单个数字人的回复风格和互动边界。

## 2. 当前不是模型微调

第 7 阶段的“调教”不是真正模型训练，不做 finetune、LoRA、训练集生成或模型权重更新。当前实现是 Prompt-level customization：把用户配置保存到 `character_settings`，聊天时由 PromptBuilder 注入 `settings_prompt`，再交给 MockLLMProvider 或后续真实 LLM Provider 使用。

没有配置 API Key 时，系统仍使用 MockLLMProvider。mock 回复只是规则测试回复，不代表真实模型理解。

## 3. CharacterSettings 字段说明

- `settings_id`：配置业务 ID
- `character_id`：所属角色 ID，一名角色对应一份配置
- `display_name`：角色显示名，默认使用角色名
- `user_persona_name`：用户扮演身份，可覆盖原角色中的用户身份
- `nickname_for_user`：角色对用户的称呼
- `relationship_with_user`：关系设定覆盖
- `relationship_stage`：关系阶段覆盖
- `tone_style`：语气风格，支持 `gentle / cold / playful / mature / yandere_light / poetic / original`
- `reply_length`：回复长度，支持 `short / medium / long`
- `intimacy_mode`：亲密模式，支持 `low / normal / high`
- `spoiler_protection`：剧透保护开关
- `pet_enabled`：是否显示 Q版桌宠
- `pet_position`：桌宠位置，支持 `bottom_right / bottom_left / floating`
- `personality_override`：性格微调
- `speaking_style_override`：说话风格微调
- `custom_prompt_notes`：用户自定义补充设定
- `forbidden_topics`：禁止话题，可用换行或逗号分隔

## 4. 每个设置如何影响 Prompt

PromptBuilder 会读取 Character、CharacterProfile、KnowledgeChunk、长期记忆和 CharacterSettings。CharacterSettings 会生成 `settings_prompt`，并加入本轮 Prompt：

- 语气风格影响角色回复的表达方式
- 回复长度影响回答详略
- 用户称呼影响角色对用户的称呼
- 关系设定和关系阶段优先覆盖原 Character 的关系字段
- 性格和说话风格微调会补充到系统角色设定
- 自定义补充设定用于用户个性化偏好
- 禁止话题要求角色温和回避
- 剧透保护开启时，真相、凶手、核心秘密必须保持悬念

配置优先级是：用户调教配置优先，但不能违反上传剧本资料、CharacterProfile 防剧透规则和 hidden / restricted 知识片段限制。

## 5. 如何影响 MockLLMProvider

MockLLMProvider 会轻量读取 `tone_style`、`reply_length`、`nickname_for_user`、`spoiler_protection`：

- `gentle` 更温柔
- `cold` 更克制
- `playful` 更俏皮
- `mature` 更像成熟姐姐
- `short` 返回更短回复
- `long` 返回稍长回复
- `nickname_for_user` 存在时优先用该称呼
- `spoiler_protection=true` 时追问真相、凶手、秘密仍保留悬念

## 6. 如何影响真实 LLM

真实模型接入时不需要改 Chat Engine 主链路。OpenAI-compatible 或其他 provider 会收到包含 `settings_prompt` 的 prompt bundle。真实模型主要通过这段配置理解用户对角色的调教要求。

## 7. 剧透保护和 forbidden_topics

`spoiler_protection` 面向剧本真相和核心秘密，优先级高于语气风格。即使用户把语气调得亲密，也不能直接泄露 hidden / restricted 内容。

`forbidden_topics` 面向用户个人边界。命中禁止话题时，角色应温和回避，并把话题带回角色关系或当前剧情。

## 8. Q版桌宠显示配置

`pet_enabled=false` 时聊天页隐藏 Q版桌宠。`pet_position` 用于控制桌宠显示位置，当前支持右下角、左下角和居中悬浮。

## 9. 后续升级方向

后续可以在保持 `CharacterSettings` 数据结构的前提下升级：

- 使用真实 LLM 分析用户偏好并推荐设置
- 把用户配置转为更精细的系统 Prompt 模板
- 结合长期记忆自动微调关系阶段
- 在真正有训练需求时，再设计 finetune / LoRA / 用户专属模型流程

这些都不属于当前第 7 阶段范围。
