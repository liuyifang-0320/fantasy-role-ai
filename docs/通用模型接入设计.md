# 通用模型接入设计

## 1. 为什么不能只写死 OpenAI

项目未来可能在不同地区上线，模型接入策略需要兼顾可替换性、成本、可用性和合规边界。只写死单一厂商，会让后续切换模型服务时牵动聊天链路、配置系统和部署流程。

## 2. 为什么需要 Multi-LLM Provider

第 4 阶段把模型调用抽成统一 Provider 层，让 Chat Engine 只依赖标准输出，不依赖具体厂商：

- `mock`
- `openai_compatible`
- `custom_http`

这样既能兼容 OpenAI，也能通过 `base_url / api_key / model` 切换 DeepSeek、通义千问 / DashScope、Kimi / Moonshot、智谱 GLM、豆包或其他兼容服务。

## 3. 三类 Provider 的区别

### `MockLLMProvider`

- 默认兜底方案
- 不依赖外部网络
- 用本地规则生成角色化回复
- 未配置 `LLM_API_KEY` 时会使用它
- 它只是规则测试回复，不代表真实模型理解剧本

### `OpenAICompatibleProvider`

- 使用官方 `openai` Python SDK
- 通过环境变量接收 `api_key`、`base_url`、`model`
- 默认使用 `chat_completions`
- 适合兼容 OpenAI 请求格式的模型服务
- 只有配置 `LLM_PROVIDER`、`LLM_API_KEY`、`LLM_BASE_URL`、`LLM_MODEL` 后才会尝试调用真实模型

### `CustomHTTPProvider`

- 预留给非 OpenAI 格式的厂商 API
- 当前阶段仅保留骨架
- 后续可按具体协议继续适配请求体和响应解析

## 4. 环境变量说明

- `LLM_PROVIDER`
- `LLM_API_KEY`
- `LLM_BASE_URL`
- `LLM_MODEL`
- `LLM_API_STYLE`
- `ENABLE_PROMPT_DEBUG`
- `LLM_TIMEOUT_SECONDS`

示例见项目根目录 `.env.example`。

## 5. `.env.example` 说明

`.env.example` 只放示例，不放真实 Key。实际部署时应复制为 `.env` 或使用部署环境注入变量，并确保 `.env` 不进入版本库和压缩包。

## 6. OpenAI-compatible API 接入方式

当服务支持 OpenAI-compatible 协议时，只需：

1. 设置 `LLM_PROVIDER=openai_compatible`
2. 设置 `LLM_API_KEY`
3. 设置 `LLM_BASE_URL`
4. 设置 `LLM_MODEL`

默认请求风格为 `chat_completions`，便于兼容国内常见模型服务。若服务支持新版 `responses`，也可以通过 `LLM_API_STYLE=responses` 切换。

## 7. 国内模型服务如何切换

国内上线时，可优先选择兼容 OpenAI 请求格式的服务，然后仅通过环境变量切换：

- `base_url`
- `api_key`
- `model`

Chat Engine、PromptBuilder、前端协议都不需要重写。

## 8. fallback 策略

以下情况自动回退到 `MockLLMProvider`：

- Provider 配置不完整
- 未配置 `LLM_API_KEY`
- 真实 Provider 调用异常
- 模型返回空文本
- `custom_http` 仍未接入具体协议

接口会返回 `provider_debug`，让前端和调试端知道是否发生了回退。

没有 API Key 时，系统不会调用真实模型，也不应在 UI 或文档里暗示已经使用真实 AI。此时的回复只用于流程测试。

## 9. provider_debug 说明

`provider_debug` 包含：

- `configured_provider`
- `provider`
- `model`
- `base_url_configured`
- `api_key_configured`
- `fallback`
- `fallback_reason`

这些字段只暴露“是否已配置”，不暴露真实 Key。

## 10. prompt_debug 开关说明

`ENABLE_PROMPT_DEBUG=true` 时返回 Prompt 预览；关闭后，接口中的 `prompt_debug` 为 `null`。正式上线前可以关闭，以减少内部 Prompt 暴露。

## 11. API Key 安全注意事项

- 不要把真实 API Key 写入代码
- 不要把 `.env` 提交到仓库
- 不要把 `.env` 打进压缩包
- 后端接口只返回是否配置，不返回真实 Key
- 日志和报错中也不应输出敏感配置
