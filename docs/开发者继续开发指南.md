# 开发者继续开发指南

## 工程目录说明

- `backend/`：FastAPI 后端
- `frontend/`：UniApp 前端，支持 H5 和 mp-weixin 构建
- `docs/`：设计文档、API 文档、交付文档
- `tests/`：自动化验收脚本和测试数据
- `scripts/`：本地开发和测试辅助脚本

## 后端模块说明

- `backend/app/api/`：路由层
- `backend/app/models/`：SQLAlchemy 数据模型
- `backend/app/schemas/`：Pydantic 请求 / 响应模型
- `backend/app/services/`：业务服务和 Provider 抽象层
- `backend/app/core/`：配置、日志等核心能力
- `backend/app/db/`：数据库 session 与初始化

## 前端模块说明

- `frontend/src/pages/`：页面
- `frontend/src/components/`：组件
- `frontend/src/api/index.js`：统一 API 封装
- `frontend/src/config/env.js`：前端 baseURL 与平台配置
- `frontend/src/utils/upload.js`：H5 / 小程序上传封装

## 新增接口应该放哪里

新增业务接口优先放在 `backend/app/api/` 对应模块中，并在 `backend/app/api/router.py` 注册。请求 / 响应结构放到 `backend/app/schemas/`。

## 新增模型应该放哪里

新增 SQLAlchemy 模型放到 `backend/app/models/`，并在 `backend/app/models/__init__.py` 导出。旧 SQLite 字段兼容逻辑放在 `backend/app/db/init_db.py`。

## 新增页面应该放哪里

新增页面放到 `frontend/src/pages/`，并在 `frontend/src/pages.json` 注册路由。API 调用统一放到 `frontend/src/api/index.js`。

## Provider 抽象层说明

当前 Provider 包括：

- `llm_provider.py`
- `ocr_provider.py`
- `image_generation_provider.py`
- `storage_provider.py`

新增真实厂商接入时必须：

- 从环境变量读取配置
- 不写死 API Key
- 调用失败可 fallback mock / local
- 不在日志中输出密钥

## PromptBuilder / ChatEngine 说明

`ChatEngine` 负责聊天链路编排：读取角色、Profile、知识库、关系、记忆、调教、安全检查，调用 Provider 并落库。

`PromptBuilder` 负责组装：

- system_prompt
- context_prompt
- knowledge_prompt
- memory_prompt
- relationship_prompt
- settings_prompt
- safety_prompt
- user_prompt

不要在 Provider 中直接拼散乱 Prompt。

## 如何避免破坏 project_id / user_id 隔离

新增数据模型时应优先考虑是否需要 `user_id` 和 `project_id`。新增查询接口时必须：

1. 获取 current_user。
2. 校验资源 `user_id`。
3. 项目内资源继续校验 `project_id`。
4. 不允许跨用户、跨项目读取知识片段、聊天、记忆、桌宠资产。

旧数据 `user_id=null` 仅为开发兼容，上线前需要迁移或关闭访问。

## 如何运行测试

先启动后端：

```bash
cd backend
python -m uvicorn main:app --host 127.0.0.1 --port 8000
```

再运行：

```bash
python tests/run_backend_smoke.py
python tests/run_project_isolation.py
python tests/run_safety_checks.py
```

## 新阶段开发前必须先跑哪些测试

建议每次新阶段开发前后都运行：

```bash
python tests/run_backend_smoke.py
python tests/run_project_isolation.py
python tests/run_safety_checks.py
npm run build:h5
npm run build:mp-weixin
```

打包前再运行：

```bash
python tests/run_package_check.py --zip <交付包.zip>
```
