# 文件解析模块设计

## 1. 当前支持的文件类型

- TXT
- DOCX
- PDF
- PNG / JPG / JPEG

其中：

- TXT / DOCX / PDF 已接入真实文本解析
- 图片当前仅保留 OCR 接口与 mock 返回

## 2. TXT 解析方式

1. 读取原始字节
2. 优先尝试 `utf-8`
3. 如果失败，再尝试 `gbk`
4. 解析后的完整文本写入 `raw_text`

## 3. DOCX 解析方式

1. 使用 `python-docx`
2. 按段落遍历文档
3. 提取非空段落文本
4. 以换行符拼接为完整文本

## 4. PDF 解析方式

1. 使用 `pymupdf`
2. 按页提取文本
3. 每页保留页码标记，例如：

```text
[第1页]
正文内容
```

4. 多页之间保留空行，便于后续分段处理

## 5. 图片 OCR 当前 mock 策略

当前阶段暂不引入真实 OCR 引擎。

对 PNG / JPG / JPEG：

- 返回 mock 文本：`当前图片 OCR 已预留，后续将接入真实 OCR。`
- `parse_status` 标记为 `mock_ocr`

这样可以先把接口、数据结构和前后端展示链路跑通。

## 6. 后续如何接入真实 OCR

后续可选择：

1. 接入 PaddleOCR
2. 接入云 OCR 服务
3. 增加图片预处理、版面分析和批量识别
4. 将识别后的文本继续写入 `parsed_documents`

## 7. 解析结果如何服务于角色生成

当前阶段：

- 解析结果会落库到 `parsed_documents`
- 角色生成前会检查上传文件是否已经解析
- 如果未解析，会自动补做解析
- 角色生成仍为 mock，但会轻微参考 `text_preview`

后续阶段：

- `raw_text` 可继续作为 RAG、角色抽取、关系抽取和真实 LLM 生成的输入基础
