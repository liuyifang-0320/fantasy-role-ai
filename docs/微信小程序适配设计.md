# 微信小程序适配设计

## 目标

第 15 阶段只做微信小程序运行骨架，不做正式上线审核、不做支付、不做正式微信登录。目标是在保留 H5 能力的基础上，让 UniApp 前端可以编译到 `mp-weixin`，并为后续微信开发者工具调试、微信登录和小程序上传文件接入留好位置。

## H5 与小程序 baseURL

前端通过 `frontend/src/config/env.js` 统一管理 API 地址，不在页面里散落后端地址。

- H5 本地开发默认：`http://127.0.0.1:8000`
- 微信小程序开发默认占位：`http://192.168.x.x:8000`
- 小程序真机或微信开发者工具调试时，需要把占位地址替换为开发机局域网 IP，或后续正式 HTTPS 域名。

当前不会写死真实生产域名。正式上线前，小程序后台必须配置合法 HTTPS request/uploadFile 域名。

## API 请求适配

`frontend/src/api/index.js` 统一读取 baseURL，并为所有请求自动带上：

```http
X-User-Id: dev_user
```

如果本地 storage 中已经有 dev 用户 ID，则优先使用该用户。当前仍是开发环境 `dev_mock` 用户体系，不是正式微信登录。

## 小程序上传兼容策略

`frontend/src/utils/upload.js` 提供统一入口：

```js
chooseAndUploadFile({ projectId })
```

H5 端优先使用 `uni.chooseFile`，微信小程序端优先使用 `uni.chooseMessageFile`，并在需要时 fallback 到 `uni.chooseImage`。上传统一走 `uni.uploadFile`，并附带：

- `project_id`
- `X-User-Id`

支持的资料类型仍由后端 `ALLOWED_UPLOAD_TYPES` 控制，默认包括 `pdf / docx / txt / png / jpg / jpeg`。小程序平台对文件来源、扩展名、大小和域名有额外限制，正式上线时需要结合微信后台配置验证。

## dev_mock 与微信登录

当前用户体系仍使用 `dev_mock`：

- 没有用户 ID 时默认使用 `dev_user`
- 用户中心可以创建和切换开发测试用户
- 所有业务数据仍按 `user_id` 隔离

第 15 阶段只新增微信登录预留按钮和后端预留接口，不调用微信真实接口，不伪造 `openid`。

## 微信登录预留接口

`POST /api/auth/wechat-login` 当前返回：

```json
{
  "status": "not_implemented",
  "message": "微信登录接口已预留，正式上线阶段接入 code2session。"
}
```

后续正式接入时，需要使用小程序 `wx.login` 获得 code，并在后端通过微信 `code2session` 换取 `openid / unionid`。不要把 `WECHAT_APPID`、`WECHAT_SECRET` 或任何真实密钥提交到仓库或压缩包。

## manifest 与 appid

`frontend/src/manifest.json` 中的小程序 appid 使用占位值：

```json
"appid": "wx_placeholder_do_not_use"
```

正式开发时需要替换为微信公众平台真实小程序 appid，并通过部署环境管理相关密钥。当前压缩包不包含真实 appid、secret 或生产域名。

## 小程序端桌宠限制

当前 Q 版桌宠是页面内组件，不是系统级手机桌面悬浮窗。微信小程序不支持真正覆盖系统桌面的常驻悬浮宠物。本阶段只保证页面内 PetWidget 尽量使用 UniApp 兼容写法，不直接依赖 `window` 或 `document`。

## 微信开发者工具运行方式

1. 进入前端目录：

```bash
cd frontend
npm install
npm run build:mp-weixin
```

2. 打开微信开发者工具。
3. 导入编译输出目录：

```text
frontend/dist/build/mp-weixin
```

4. 如果需要连接本地后端，把 `frontend/src/config/env.js` 中的小程序 baseURL 占位地址改为开发机局域网 IP，例如 `http://192.168.1.23:8000`。

## 正式上线前清单

- 注册并配置微信小程序账号。
- 配置真实小程序 appid。
- 准备 HTTPS 后端域名，并在微信后台配置 request/uploadFile 合法域名。
- 接入正式微信登录 `code2session`。
- 准备隐私协议、用户协议和数据删除机制。
- 配置内容安全审核与敏感内容策略。
- 使用生产数据库、对象存储和正式日志监控。
- 关闭或迁移开发阶段的 `legacy_null_user` 兼容策略。
